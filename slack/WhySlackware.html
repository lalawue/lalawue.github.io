为什么选择 Linux？

这里主要讨论的是桌面系统（PC）。在这方面，Windows 已经很好，很容易“上手”，特别游戏支持方面没得说。虽说 Linux 也不错，可是 Linux 的起点比较高，可能需要蛮长的一段时间才能让你有“上手”的感觉，而在 Windows 下面就不会这样。

不过 Linux，一种类 Unix 操作系统，会带给你另一种与使用 Windows 不同的感觉……

当你选择使用 Linux，是不是考虑过下面的几点1:

  • 你需要大幅削减你的 IT 支出（比如：削减 90% 的软件花费，不需要每次升级软件的时候升级你的硬件，等等）。
  • 你需要大幅提高你的 IT 质量（比如：更低的维持和管理花费，提高效率及有效性）。
  • 你需要一个可以信任的系统，被担保且没有后门、跟踪系统以及其他的安全方面的危险。

或者，考虑一下关于个人方面的更多的原因：

  • 你希望或者需要提高自己的 Unix 技能，却无力承担购买一个 Unix 工作站。
  • 你希望学习网络技术或者软件开发技能，并使用一些现实世界的工具。
  • 你有机会援救一些“老”电脑，并使用它们做一些有用的事。
  • 你想建立一个家庭网络，允许你的家庭成员同时使用互联网。

考虑以上几点，看看是不是真的需要学习、使用 Linux。如果你刚开始学习使用 Linux，注意，它多多少少会花掉你宝贵的时间。万事开头难……

但一旦你做了决定，有了自己的想法，后面的弯路就少了很多。

为什么选择 Slackware？

" Stable, Solid, Simple, Sensible -- the 4S rule."2

这也是 Linuxsir 上 Slackware 社区的招牌，:)

Slackware 是稳定的。Slackware 只有一个人维护，他就是 Patrick Volkerding，具有 10 年以上的发行版维护经验，特别注重系统的稳定性。

Slackware 是稳固的。由于 Patrick 的努力，Slackware 的臭虫报告和安全问题比其他发行版少得多。

Slackware 是简单的。简单意味着没有不必要的复杂，可以很容易地配置和管理，对出现的问题和需求可以立刻回应。

Slackware 是合理的。Slackware 紧贴标准，可以为用户和管理员减少了很多麻烦，看看下面的 KISS 原则。

KISS 原则

什么是 KISS ？当然 KISS 有着许多不同的说法，不过在这里我指的是“Keep it Simple, Stupid”，这是 Unix 的设计哲学，也是 Slackware 的设计哲学。它贯穿 Slackware 的始终3：

  • 使用 tgz 包管理，与 deb 和 RPM 不同，没有依赖性检查，但“ tgz 对于有经验的用户来说，比 RPM 更为强大，并避免了 RPM 之类管理器的依赖性问题。”
  • 使用BSD 风格的 init 脚本，容易调试和使用（个人观点）。详细的区别请看<< SysV 和 BSD 启动风格的比较(中英文对照)>>
  • 基于文本界面的配置工具，“让你感觉到实实在在的操作 Linux，而不是被图形界面包裹起来的东西，因此 Slackware 让你对 Linux 的理解更为直接了当，更具普遍性。”

一些个人观点

这里的引用来自 Sir 上 Dasn 的帖子，自从使用 Slackware 一段时间后，我很是赞同。

Patrick has ten years experience on the development of Linux distro,
so we trust him.

稳定是我们一直追求的，我们相信 Patrick，相信他的经验，:)

Personally, for my own comparison, Slackware provides us with the 
most effective way to build a full functional software platform.

我们需要一个能快速建立的好用的系统，遵循 KISS 原则的 Slackware 正好满足你。
